📦 WHAT YOU HAVE
================

A complete, production-ready design package for building autonomous data 
pipeline optimization systems. Agent-driven exploration running 50-100 
experiments/night. Every change tracked. Improvements compound.

🎯 WHAT IT SOLVES
==================

• 70% of data engineering time = manual pipeline tuning
• No systematic way to compare optimization approaches  
• Scaling becomes chaotic (1 pipeline → 100+ pipelines)

💡 HOW IT WORKS
================

1. Human defines boundaries (program.md: what agent can/cannot do)
2. Agent autonomously proposes & tests improvements (modifies ONLY analysis.py)
3. Engine runs experiment with fixed time/cost budget
4. Metric improves? → keep commit. Worse? → discard & try next idea.
5. Repeat 50-100 times overnight while you sleep.

🏗️ ARCHITECTURE IN 30 SECONDS
============================

Agent (Claude) → Proposes improvement → Modifies analysis.py → Git commit
↓
Engine (executor) → Runs with fixed budget → Collects metrics → Compares
↓
Azure Services (Synapse, ADLS, Insights) → Execute pipeline → Track results
↓
Results.tsv → If better, keep; if worse, discard → Repeat 50-100x/night

🔧 TECHNOLOGY STACK
===================

• Agent:          Claude API (autonomous loop)
• Executor:       Python 3.10+ (subprocess isolation)
• Cloud:          Azure (Synapse SQL, ADLS Gen2, App Insights)
• Tracking:       Git + TSV (audit trail)
• Deployment:     Docker + Azure Container Instance

❓ FAQ
======

Q: Will the agent break production?
A: No. Agent modifies test code only (analysis.py). Production pipelines separate.

Q: How much does it cost?
A: ~$800/month infrastructure + ~$2-3K/month for experiments = offset by gains.

Q: Can we scale to 100+ pipelines?
A: Yes. Start with 1, prove it, then expand (roadmap included).

Q: What if agent gets stuck?
A: Monitored daily. If issues → interrupt + debug (30 min typical).

Q: Do we need new Azure skills?
A: No. Design uses standard Azure services (Synapse, ADLS, Insights).
